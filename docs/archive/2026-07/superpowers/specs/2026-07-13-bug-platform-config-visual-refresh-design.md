# Bug 平台配置区视觉优化设计

## 背景

Bug 工单页的平台配置区当前把平台信息、登录、机器人映射、同步操作和 Hook URL 放在同一视觉平面。页面级样式又把普通、次要、图标和危险按钮统一到至少 44px，导致桌面端控件偏大，操作层级不清晰。新增平台和移除机器人使用文本 `+`、`×` 的大方形按钮，删除平台与保存配置视觉重量接近；右上角“收起配置”沿用实心高亮样式，在实际页面状态下出现文字与背景对比不足。

## 目标

- 将配置面板整理为便于扫描的紧凑分区，降低桌面端纵向占用。
- 明确主要、次要、图标和危险操作的视觉层级。
- 保证“平台配置 / 收起配置”在默认、悬停、聚焦和展开状态下均清晰可读。
- 用稳定的 SVG 图标替代新增和移除操作中的字体字符。
- 补齐可见表单标签、键盘焦点、禁用态对比和移动端触控尺寸。
- 保持现有保存、登录、同步、删除、机器人映射和 Hook URL 数据流不变。

## 方案选择

采用分区紧凑布局，而不是只换皮或分步配置：

- 只换皮改动小，但不能解决信息混杂和操作层级问题。
- 分步配置最清爽，但会增加操作路径，不适合需要快速核对多项配置的排障工作台。
- 分区紧凑布局在同屏可见性和可读性之间更平衡。

## 页面结构

### 配置入口

页面右上角保留一个紧凑的配置入口，包含设置 SVG、文字和方向箭头：

- 收起时为蓝色实心按钮，文案“平台配置”，箭头向下。
- 展开时为浅蓝背景、深蓝文字，文案“收起配置”，箭头向上。
- 两种状态均保留明确的悬停和 `:focus-visible` 反馈，不使用白底白字。
- 使用 `aria-expanded` 和 `aria-controls` 表达折叠状态。

### 平台信息区

- 顶部显示可换行的平台标签列表；当前平台使用浅蓝选中态，同时保留 `aria-pressed`。
- “新增平台”改为加号 SVG +“新建平台”的紧凑次级按钮，不再显示孤立的大方块。
- 名称、类型、地址、登录方式使用可见标签和紧凑控件。
- 登录状态显示为“已登录 / 未登录”状态徽标；登录和清除登录态使用次级按钮。
- “启用平台”使用紧凑开关视觉，但继续使用原生 checkbox 和 label。

### 排障机器人区

- 区块标题、说明和“添加机器人”放在同一工具栏；添加入口为小号次级按钮，包含加号 SVG。
- 每个机器人行按“名称与路径 / 环境 / 移除”排列。
- 移除按钮改为垃圾桶 SVG 幽灵按钮，默认弱化，悬停和聚焦时显示浅红背景与深红图标；保留 `aria-label="移除机器人"`。
- 机器人选择器、搜索和添加行为保持不变。

### 同步与接入区

- 定时同步开关和间隔输入组成一组；关闭定时同步时，间隔输入保持现有禁用逻辑。
- 主动同步、Bug ID 输入和“拉取指定 Bug”组成紧凑操作行。
- Hook URL 使用弱背景代码框展示，复制操作改为紧凑次级按钮。

### 面板底部操作栏

- 左侧放小号红色文字按钮“删除平台”，与保存动作保持足够距离；仍使用现有确认对话框。
- 右侧放唯一主要动作“保存配置”。
- 删除和保存不再并排呈现为同等重量的大按钮。

## 视觉规则

- 桌面端普通按钮和表单控件的视觉高度为 36–38px；图标按钮保持 40px 左右的独立点击区域。
- 640px 以下普通控件恢复至少 44px，并允许主要操作占满可用宽度。
- 区块使用浅灰背景、细边框和 8px 圆角；不增加装饰性渐变或厚重阴影。
- 主要按钮使用现有蓝色；次级按钮使用白底深灰字；危险文字使用深红色；禁用态使用灰底和可读的灰色文字。
- 所有图标使用统一的 24×24 viewBox，视觉尺寸 16–18px。
- 颜色和阴影过渡为 150–200ms；`prefers-reduced-motion: reduce` 时取消过渡。

## 响应式设计

- 900px 以下平台信息字段与同步操作切为单列或双列，不产生横向滚动。
- 640px 以下区块标题工具栏允许换行，普通文字按钮占满可用宽度，图标按钮保持独立。
- 机器人名称和路径允许换行；环境选择和移除按钮不会挤出容器。
- 保留页面现有 375、768、1024、1440px 响应式契约。

## 状态与错误处理

- 不改变任何 API 调用、加载状态、toast、确认对话框或数据结构。
- 保存、登录、清除登录态、同步和拉取按钮继续使用现有禁用条件。
- 禁用态不以颜色作为唯一信息：同时保留原生 `disabled` 属性和不可用光标。
- 删除平台继续二次确认，默认操作仍为取消。

## 实现边界

- 主要修改 `web/src/pages/BugInboxPage.vue` 和对应测试。
- 不抽取新的平台配置业务组件，避免在本次视觉优化中引入大量 props/emits 重构。
- 不修改 Bug 收件箱、工单详情、故障闭环或后端接口。
- 不改变全局按钮设计系统；新样式限定在平台配置区，避免影响其他页面。

## 测试与验收

- 先添加失败测试，覆盖折叠按钮双状态、`aria-expanded`、三个分区和可见标签。
- 验证新增平台、添加机器人、移除机器人均使用 SVG，不再依赖 `+`、`×` 字符。
- 验证删除入口位于操作栏左侧、保存入口位于右侧，并保留原有删除确认和保存行为。
- 验证登录、同步、机器人环境选择和 Hook URL 复制行为未回归。
- 验证 375、768、1024、1440px 的结构标记和无横向溢出约束。
- 运行 BugInboxPage 定向测试、Web 全量测试、生产构建和 lint。

验收时应看到：配置入口文字始终可读，配置内容分区明确，桌面控件明显更紧凑，新增/移除/删除操作不再抢夺主要动作的注意力，移动端仍具有足够的点击区域。
