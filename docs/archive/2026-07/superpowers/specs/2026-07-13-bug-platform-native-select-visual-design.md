# Bug 平台原生下拉框视觉优化设计

## 背景

上一轮调整扩大了 Bug 平台配置区的列宽和控件最小高度：平台类型列最小宽度改为 200px，机器人环境列改为 240–280px，桌面控件高度改为 40px。构建产物和页面截图均证明这些尺寸已经生效。

但页面仍直接显示浏览器原生 `select` 外观，系统箭头、白底和边框状态与调整前几乎一致，导致用户无法感知到明显优化。根因不是部署或 CSS 未生效，而是上一轮方案只处理了布局尺寸，没有处理下拉框本体的视觉层。

## 目标

- 让平台配置区的下拉框在截图和实际操作中具有明显、统一的视觉表现。
- 保留原生 `select` 的键盘操作、系统选项弹层和无障碍语义。
- 与现有输入框、按钮、颜色变量和响应式规则保持一致。
- 将改动严格限制在 Bug 平台配置区，不影响其他页面或普通表单控件。

## 非目标

- 不实现自定义下拉菜单、搜索、虚拟列表或多选。
- 不增加 JavaScript 展开状态、键盘事件或浮层定位逻辑。
- 不修改平台类型、登录方式、机器人环境的数据绑定和选项来源。
- 不再次调整已经确认的平台类型和机器人环境列宽。

## 视觉方案

### 基础外观

仅对 `.platform-config select.form-control` 应用自定义视觉壳层：

- 使用 `appearance: none` 和 `-webkit-appearance: none` 隐藏浏览器默认箭头。
- 通过 CSS `background-image` 放置统一的内联 SVG 下箭头，不增加模板节点。
- 箭头使用中性深灰色，尺寸约 16px，固定在右侧 12px 居中位置。
- 文本左内边距为 12px，右内边距为 40px，避免文字与箭头重叠。
- 保持现有圆角、字体和白色表面背景，使用项目已有设计变量。
- 增加 `cursor: pointer`，明确这是可交互控件。

### 尺寸

- 桌面高度继续使用平台配置局部变量 `--config-control-height: 40px`。
- 640px 及以下继续使用 44px 触控高度。
- 平台类型列继续保持最小 200px。
- 机器人环境列继续保持 240–280px。
- 不改变输入框、按钮或配置区外下拉框的尺寸。

### 交互状态

- 默认：白色背景、现有灰色边框、中性深色文本和灰色箭头。
- 悬停：边框切换为浅蓝色；不改变尺寸，不产生布局跳动。
- 聚焦：保留现有可见蓝色焦点环，并使用蓝色边框强化当前控件。
- 禁用：沿用配置区现有灰底、灰字和 `not-allowed` 光标；箭头保持可辨识但降低视觉强调。
- 状态切换使用 150–200ms 的颜色过渡；不添加位移、缩放或旋转动画。
- `prefers-reduced-motion: reduce` 下不依赖动画表达任何状态。

## 可访问性

- 模板继续使用原生 `select`、`option` 和现有 `label` 关联。
- 不拦截浏览器原生键盘行为，Tab、方向键、Enter/Space 行为保持不变。
- 焦点状态不能只依赖颜色变化，继续保留 2px 外轮廓。
- 移动端触控目标不小于 44px。
- 自定义箭头为 CSS 装饰背景，不进入无障碍树，也不产生重复朗读。

## 响应式与作用域

- 样式限定为 `.platform-config select.form-control`，避免影响向导、日志、故障闭环等页面。
- 现有 1200px 两列、900px 单列和 640px 触控尺寸规则保持不变。
- 下拉框始终使用 `width: 100%` 和 `min-width: 0`，避免窄屏横向滚动。
- 右侧箭头使用固定背景位置，不依赖列宽，因此平台类型和机器人环境两类下拉框表现一致。

## 测试与验证

### 自动化测试

在 `BugInboxPage.test.ts` 的源码 CSS 契约测试中增加以下断言：

- 平台配置下拉框包含 `appearance: none` 和 `-webkit-appearance: none`。
- 包含 SVG `background-image`、固定背景位置和背景尺寸。
- 左右内边距为 12px/40px。
- 包含 `cursor: pointer`、悬停边框和聚焦边框规则。
- 现有禁用状态、桌面 40px、移动端 44px、列宽和断点断言继续通过。
- 选择器必须带 `.platform-config` 作用域，防止样式泄漏到其他页面。

### 验证命令

```bash
npm --prefix web test -- --run src/pages/BugInboxPage.test.ts
npm --prefix web test -- --run
npm --prefix web run build
make lint
git diff --check
```

### 视觉验收

- 平台类型、登录方式和机器人环境三个下拉框使用同一箭头、内边距和状态样式。
- 375px、768px、1024px、1440px 下无横向滚动或文字与箭头重叠。
- 鼠标悬停、键盘聚焦和禁用状态可清楚区分。
- 与同一配置区输入框相比，下拉框高度、圆角、字体和边框体系一致。
