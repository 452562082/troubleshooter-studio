# Bug 收件箱紧凑工具栏与同步接入布局设计

## 背景

Bug 收件箱左栏当前把“同步我的 Bug”按钮绝对定位在列表面板右上角，同时通过固定 `padding-right: 112px` 给标题避让。按钮实际宽度受中文文案、字体和缩放影响，可能超过固定预留空间，从而与“1 条”等数量信息重叠。

“同步与接入”区域则把后台定时同步与“拉取指定 Bug”拆成两整行。桌面宽屏下手工拉取输入框占据整行，信息密度低，也割裂了两个同步入口的关系。

## 目标

- 消除收件箱数量与同步按钮在任何支持宽度下的重叠。
- 将后台定时同步与手工拉取组织为同一组控制区，并用可见分隔组件区分。
- 保持同步、手工拉取、Hook URL 的现有行为、禁用条件和数据作用域不变。
- 保持键盘可达、可见焦点和移动端至少 44px 的触控目标。
- 在 375px、768px、1024px、1440px 下不产生横向滚动。

## 根因

### 收件箱标题重叠

`BugInboxPage.vue` 将同步按钮作为 `.ticket-list-panel` 的绝对定位子元素，而 `BugTicketList.vue` 的标题和数量仍通过 `justify-content: space-between` 自行布局。两者不共享同一个布局上下文，固定右内边距无法表达按钮的真实宽度，因此在缩放、字体变化或较窄面板下会碰撞。

### 手工拉取占满整行

`.sync-settings` 与 `.manual-bug-row` 是两个独立直属行。后者使用 `minmax(0, 1fr) auto`，会自然占满容器宽度；这不是输入框本身的尺寸问题，而是分组层级不合理。

## 设计

### 1. 收件箱正常流工具栏

同步入口仍由 `BugInboxPage.vue` 持有，因为它依赖页面级的平台列表和同步状态。列表组件新增一个具名 `actions` slot，并把该 slot 放入自己的 `.list-heading` 正常流中。

标题栏的语义与视觉顺序为：

1. “Bug 收件箱”标题；
2. Bug 数量；
3. 弹性空白；
4. “同步我的 Bug”按钮。

标题、数量和按钮共同参与网格或 flex 布局，不再使用绝对定位，也不再使用固定 `padding-right` 猜测按钮宽度。按钮继续使用现有 SVG、`data-action="sync-enabled-platforms"`、加载文案、旋转状态和禁用条件。

桌面标题栏允许数量靠近标题，操作按钮靠右。可用宽度不足时，操作区可换到下一行，但不得覆盖标题或数量。

### 2. 同步与接入组合控制行

将现有 `.sync-settings` 与 `.manual-bug-row` 包进一个 `.sync-control-row`，形成三列：

```text
[后台定时同步 + 间隔]  [分隔线]  [指定 Bug 输入框 + 拉取按钮]
```

- 左侧 `.sync-settings` 按内容宽度布局，保留开关和分钟输入。
- 中间使用独立 `.sync-control-divider` 视觉组件；桌面为竖线，标记 `aria-hidden="true"`，不进入辅助技术语义树。
- 右侧 `.manual-bug-row` 使用 `minmax(0, 1fr) auto`，输入框消费剩余宽度，按钮按内容宽度。
- Hook URL 保持独立下一行，仍位于组合控制行之后。

不新增状态，不改变 `fetchManualBug` 对当前选中平台的依赖，也不改变后台轮询配置和 Hook URL 生成逻辑。

### 3. 响应式规则

- 大于 900px：组合控制行采用 `auto 1px minmax(0, 1fr)` 三列。
- 640px 至 900px：组合控制行改为单列，分隔线变为横线，避免中等宽度下开关、间隔、输入和按钮相互挤压。
- 小于等于 640px：继续单列；手工拉取按钮全宽，所有输入和按钮至少 44px 高。
- 收件箱标题工具栏在宽度不足时允许操作区整行换行，数量始终与标题保持可读间距。

## 组件边界

### `BugTicketList.vue`

- 负责标题、数量、搜索框和工单列表布局。
- 只提供 `actions` slot，不理解同步平台或同步状态。
- 没有提供 slot 内容时，现有独立使用方式保持兼容。

### `BugInboxPage.vue`

- 继续拥有同步按钮及其行为。
- 将按钮通过 `BugTicketList` 的 `actions` slot 注入标题栏。
- 负责同步与接入组合控制行和响应式分组。

这种边界避免把业务同步逻辑下沉到纯展示列表组件，同时让标题、数量和操作处于同一布局上下文。

## 可访问性与交互

- 保留同步按钮动态 `aria-label`、禁用状态和加载反馈。
- 分隔线仅用于视觉分组，使用 `aria-hidden="true"`。
- 输入框继续使用显式 `<label for>` 关联。
- 按钮保持可见 `focus-visible` 状态；移动端触控目标不低于 44px。
- 不使用颜色作为唯一状态表达，不增加仅 hover 可发现的操作。

## 测试策略

先写失败测试，再改生产代码：

1. `BugTicketList.test.ts` 验证 `actions` slot 与标题、数量处于同一 `.list-heading`，且无 slot 时兼容。
2. `BugInboxPage.test.ts` 验证唯一同步按钮位于列表标题 action 区，不再是 `.ticket-list-panel` 的绝对定位兄弟。
3. 源码布局契约验证删除 `.refresh-button { position: absolute; ... }` 和固定标题右内边距。
4. 验证 `.sync-control-row` 直属包含 `.sync-settings`、分隔组件和 `.manual-bug-row`，Hook URL 仍在其后。
5. 验证桌面三列、中屏单列/横分隔、小屏按钮全宽和 44px 触控规则。
6. 运行页面与列表组件聚焦测试、全量前端测试、TypeScript 检查和生产构建。

## 非目标

- 不修改多平台同步算法、错误聚合或本地列表刷新次数。
- 不修改后台定时同步的保存协议。
- 不移动 Hook URL 到组合控制行。
- 不改变 Bug 工单详情、列表卡片或故障闭环入口。
- 不新增后端 API 或持久化字段。

## 验收标准

- “Bug 收件箱”“N 条”“同步我的 Bug”在支持宽度和缩放下不重叠。
- 同步按钮只出现一次，行为与现有实现一致。
- 桌面端定时同步与手工拉取处于同一行，中间有明确竖向分隔。
- 中小屏自动堆叠且分隔线转为横向，不出现横向滚动。
- Hook URL 保持独立下一行。
- 相关聚焦测试、全量前端测试、TypeScript 检查和构建全部通过。
