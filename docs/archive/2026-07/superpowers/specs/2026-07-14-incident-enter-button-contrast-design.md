# “进入故障闭环”按钮对比度优化设计

## 背景与证据

故障闭环页的“进入故障闭环”是已有 Case 的主要导航操作。截图证据显示，鼠标悬停时按钮呈现白色文字和近白色背景，文案几乎不可辨认。

根因是页面级通用规则 `.btn:hover:not(:disabled)` 的选择器优先级高于 `.btn.primary`：悬停时背景被覆盖成浅色，主按钮的白色文字却被保留。

## 目标

- “进入故障闭环”在默认、悬停、键盘聚焦状态下始终清晰可读。
- 可操作态与禁用态具有不同视觉语义，不让用户误判按钮不可用。
- 保持现有蓝色选择体系和危险操作的红色层级，不调整布局或交互流程。

## 方案

采用实心蓝色主按钮：

| 状态 | 背景 / 边框 | 文字 | 其他反馈 |
|---|---|---|---|
| 默认 | `#2563eb` | 白色 | 44px 最小点击高度 |
| Hover | `#1d4ed8` | 白色 | 180ms 颜色过渡，不产生位移 |
| Focus visible | 保持当前实心蓝色 | 白色 | 清晰外圈与 2px offset |
| Disabled | 中性浅灰背景和边框 | 深灰文字 | `not-allowed` 光标，不复用白字主按钮样式 |

通过作用域更精确的主按钮 hover/disabled 规则解决选择器覆盖，不修改全局按钮系统，避免影响其他页面。

## 测试与验收

- 先增加样式回归测试，证明当前主按钮 hover 被通用浅色规则覆盖。
- 测试明确锁定默认、hover、focus-visible、disabled 的颜色和过渡契约。
- 运行 `IncidentWorkbenchPage` 聚焦测试、Web 全量测试、`vue-tsc --noEmit` 和生产构建。
- 人工复查：鼠标悬停时仍为深蓝底白字，键盘聚焦圈清楚，禁用态不与可用态混淆。

## 非目标

- 不修改“重新开始故障闭环”的危险操作样式。
- 不调整机器人选择、Case 状态或故障闭环业务逻辑。
- 不重构全局 `.btn` 设计系统。
