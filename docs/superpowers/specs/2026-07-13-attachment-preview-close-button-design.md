# 附件预览关闭按钮视觉优化设计

## 背景

Bug 工单附件预览弹窗当前复用通用 `.btn.icon` 样式，并以文本字符 `×` 表示关闭。按钮因此呈现为带明显边框的方块，视觉重量高于标题和附件内容；字符字形也会随字体变化，与弹窗的简洁风格不一致。

## 目标

- 让关闭按钮弱化为标题栏中的辅助操作，不抢占附件内容的视觉焦点。
- 保持至少 44×44 像素的鼠标和触摸点击区域。
- 保留明确的悬停、按下和键盘焦点反馈。
- 使用稳定的 SVG 图标和无障碍名称，不依赖字体中的乘号字符。

## 方案

仅调整 Bug 工单附件预览弹窗：

- 为按钮使用独立的 `attachment-preview-close` 类，不改变其他 `.btn.icon` 按钮。
- 使用 20×20 的内联 SVG 关闭图标，保持统一线宽和居中效果。
- 按钮采用 44×44 的无边框圆形幽灵样式，默认透明；悬停时显示浅灰背景并加深图标颜色。
- 按下状态使用更深一档的背景色，不通过缩放改变布局。
- `:focus-visible` 使用现有蓝色焦点环；继续保留 `aria-label="关闭附件预览"`。
- 标题过长时继续在剩余空间内换行，关闭按钮固定尺寸且不收缩。

不改变弹窗尺寸、图片展示方式、遮罩点击关闭逻辑，也不扩展到平台配置中的删除按钮。

## 验证

- 组件测试先验证现状缺少专用按钮和 SVG 图标，并观察失败。
- 实现后验证按钮具有专用类、可访问名称及 SVG 图标。
- 验证点击按钮仍会关闭附件预览。
- 运行 Bug 工单页面测试、Web 全量测试、构建和 lint。

## 验收标准

- 关闭按钮不再显示明显方形边框或字体字符 `×`。
- SVG 图标在 44×44 点击区域内水平、垂直居中。
- 悬停、按下和键盘焦点状态清晰，且不引发布局位移。
- 现有附件预览和其他图标按钮行为不受影响。
