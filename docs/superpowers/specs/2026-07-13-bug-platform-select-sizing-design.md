# Bug 平台下拉框与控件尺寸统一设计

## 背景

Bug 平台配置页采用紧凑表单布局，但当前平台类型列固定为 `150px`，机器人环境列最大为 `180px`，下拉框相对相邻输入框过窄。平台配置内的表单控件高度为 `36px`，与按钮、开关和页面通用控件的视觉基线也不统一。

## 目标

- 提升平台类型和机器人环境下拉框的可读性与可操作空间。
- 统一平台配置区域内输入框、下拉框、紧凑按钮、开关和同步间隔控件的桌面高度。
- 保持现有表单数据、事件处理、键盘操作和移动端触控规范不变。
- 在 375、768、1024 和 1440px 等常用视口避免横向溢出。

## 尺寸规范

- 平台配置桌面控件统一使用 `40px` 最小高度。
- 移动端继续使用 `44px` 最小触控高度。
- 平台类型列最小宽度为 `200px`。
- 机器人环境列宽为 `minmax(240px, 280px)`。
- 删除机器人按钮桌面保持 `40px` 独立触控目标，移动端保持 `44px`。
- 下拉框保留原生语义和键盘行为，并统一水平内边距及右侧箭头留白。

## 响应式布局

### 大于 1200px

平台名称、平台类型和平台地址保持同一行。平台名称使用弹性宽度，平台类型至少 `200px`，平台地址继续占据主要宽度。

### 901–1200px

平台信息改为两列：平台名称与平台类型位于第一行，平台地址独占第二行，防止固定最小宽度挤压内容。

### 不大于 900px

平台信息与登录区域继续使用单列布局。现有不大于 640px 的 `44px` 触控尺寸规则保持有效。

## 视觉与可访问性

- 所有下拉框继续使用可见标签，不依赖 placeholder。
- 保留现有焦点样式、禁用状态对比度和错误反馈。
- 不使用自绘 JavaScript 下拉组件，不改变原生 `select` 行为。
- 宽度和高度变化不能造成文本截断、删除按钮位移或横向滚动。

## 测试

- 验证平台配置桌面控件统一为 `40px`。
- 验证平台类型最小 `200px`、机器人环境列为 `240–280px`。
- 验证 1200px 两列布局与 900px 单列布局契约。
- 保留并运行移动端 `44px` 触控尺寸测试。
- 运行 Bug 工单页聚焦测试、全量 Web 测试、生产构建和 lint。

## 非目标

- 不修改 Bug 平台后端接口、保存逻辑或数据结构。
- 不引入第三方表单或下拉框组件库。
- 不重构平台配置页面的其他交互流程。
