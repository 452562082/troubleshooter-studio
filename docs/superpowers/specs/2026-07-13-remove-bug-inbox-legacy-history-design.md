# Bug 工单页移除旧版运行记录设计

日期：2026-07-13

## 背景与决策

`BugInboxPage` 当前在完整工单详情下方展示“历史运行记录（只读）”或“机器人上下文”。这些内容来自旧 `InvestigationRun`，与新的“故障闭环”历史 Case 展示重复，容易让用户误以为 Bug 工单页仍承载验证和排障流程。

本设计采用完全移除 Bug 工单页旧版运行记录区域的方案。它取代 `2026-07-13-bug-inbox-incident-workbench-split-design.md` 中关于在 Bug 工单页折叠展示旧 `InvestigationRun` 的要求。

## 范围

- 从 `/bugs` 页面移除“历史运行记录（只读）”和“机器人上下文”区域。
- Bug 工单页不再调用 `listBugInvestigationRuns`，也不再维护旧记录标签、Markdown 渲染或复制状态。
- 保留完整工单详情、附件预览和“进入故障闭环”操作。
- 保留后端旧数据、迁移逻辑、bridge 兼容接口和故障闭环中的 `legacy_archived` Case 展示。

## 方案比较

1. **完全移除（采用）**：信息架构最清晰，也避免无意义的前端请求；历史数据仍能在故障闭环中审计。
2. 保留弱化链接：兼容入口更多，但仍会让用户面对两套历史概念。
3. 保留折叠卡片：改动最少，但延续当前重复和歧义。

## 行为与异常

- 选择或切换 Bug 时只加载工单详情，不读取旧 runs。
- 旧 runs 接口异常不再影响 Bug 工单页，也不再产生“读取历史运行记录”提示。
- 故障闭环继续负责展示迁移后的旧记录，并提供从新一轮验证继续的操作。

## 测试

- 构造含旧 run 与 `last_context` 的 Bug，断言页面不显示两个旧版区域。
- 断言 `listBugInvestigationRuns` 从未调用。
- 保留并运行工单浏览、附件预览和跳转故障闭环的既有测试。

## 验收标准

- `/bugs` 页面不存在“历史运行记录（只读）”“机器人上下文”及其复制按钮。
- Bug 工单页不请求旧运行记录。
- `/incidents` 的历史归档能力与后端旧数据不受影响。
